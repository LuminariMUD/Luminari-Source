# 루미나리MUD 시작 가이드
## 개요
이 가이드는 소스 코드에서 루미나리MUD를 설정하고 실행하는 방법을 다룹니다.
### 필수 조건
- Linux/Unix 시스템 (Ubuntu, Debian, CentOS, WSL, macOS)
- Git 설치됨
- 종속성 설치のための 인터넷 연결
## 설치
### 자동화된 설정
대부분의 사용자에게 권장되는 접근 방식:
```bash
# 저장소 복제
git clone https://github.com/LuminariMUD/Luminari-Source.git
cd Luminari-Source
# 배포 스크립트 실행 (모든 것을 처리: 데이터베이스, 세계 데이터, 빌드)
# 주의: 설정 중 MySQL 루트 비밀번호를 입력하라는 메시지가 표시됩니다
./scripts/deploy.sh
# 서버 시작
./bin/circle -d lib
```
이제 끝났습니다! MUD 클라이언트로 `localhost:4000`에 연결하세요.
**무엇을 하는가:** 스크립트는 데이터베이스를 자동으로 설정하고, 세계 데이터를 초기화하며, 구성 파일을 복사하고, MUD를 빌드합니다. 세계 초기화는 기본적으로 활성화되어 있습니다 (서버 시작에 필요).
### 옵션과 함께 설정
배포에 더 많은 제어를 위해:
```bash
# 저장소 복제
git clone https://github.com/LuminariMUD/Luminari-Source.git
cd Luminari-Source
# 사용자 정의 옵션으로 배포 실행
./scripts/deploy.sh --dev # 디버그 심볼이 포함된 개발 빌드
# 또는
./scripts/deploy.sh --auto # 가능한 한 프롬프트 건너뜀
# 서버 시작
./bin/circle -d lib
```
배포 스크립트는 자동으로:
- 누락된 종속성을 설치하고 게임을 빌드합니다 (Autotools 우선)
- `luminari` 데이터베이스와 사용자를 프로비저닝하며, 모든 테이블(야생지, 선박, PubSub, 도움말 등)이 존재하도록 코드 내 `db_init_system` 루틴을 실행합니다
- 최소 세계 데이터(구역, 방, 몹, 객체)를 초기화합니다 - 기본적으로 활성화됨
- 생성된 자격 증명을 `lib/mysql_config`에 작성합니다 (모드 600)
**주의:** 데이터베이스와 세계 데이터가 모두 필요합니다. 스크립트는 둘 다 자동으로 처리합니다. 자격 증명을 새로 고치고 스키마를 마이그레이션하기 위해 스크립트를 안전하게 재실행할 수 있습니다.
### 수동 설정
자세한 수동 설정 지침은 [전체 배포 가이드](deployment/DEPLOYMENT_GUIDE.md)를 참조하세요.
## MUD에 연결
### MUD 클라이언트
모든 MUD 클라이언트를 사용하여 연결할 수 있습니다:
**인기 클라이언트:**
- **Mudlet** - 기능이 풍부하고, 크로스 플랫폼 (권장)
- **MUSHclient** - Windows, 강력한 스크립팅
- **TinTin++** - 터미널 기반, Unix/Linux
- **SimpleMU** - Windows, 초보자 친화적
- **Blowtorch** - Android 모바일 클라이언트
- **MUDRammer** - iOS 모바일 클라이언트
**기본 연결:**
- 호스트: `localhost` 또는 `127.0.0.1`
- 포트: `4000` (기본)
### 첫 번째 로그인
1. 첫 번째 캐릭터 생성 (자동으로 관리자 권한 부여)
2. 이름, 비밀번호, 종족, 클래스 선택
3. 캐릭터 생성 완료
4. 기본 시작 방에서 시작
## 필수 명령어
### 기본 명령어
```
look / l - 주변 환경 조사
north/south/etc - 이동 (또는 n/s/e/w/u/d)
get <item> - 아이템 집기
drop <item> - 아이템 떨어뜨리기
inventory / i - 인벤토리 확인
equipment / eq - 장착 아이템 보기
who - 온라인 플레이어 목록
say <message> - 방 안의 다른 사람들과 대화
tell <player> <msg>- 개인 메시지 보내기
help <topic> - 도움말 시스템 접근
```
### 전투 명령어
```
kill <target> - 몹/플레이어 공격
flee - 전투 탈출
rescue <player> - 누군가를 전투에서 구출
bash - 상대를 넘어뜨리기 시도
cast '<spell>' - 주문 시전
```
### 캐릭터 명령어
```
score - 캐릭터 통계 보기
affects / aff - 활성 효과 목록
practice - 스킬 보기/개선
train - 훈련 포인트 소비
levelup - 준비되면 레벨업
save - 강제 캐릭터 저장
quit - 게임 종료
```
## 빌더 명령어 (관리자)
첫 번째 생성된 캐릭터로서 빌더 명령어에 접근할 수 있습니다:
### 온라인 생성 (OLC)
```
redit - 방 에디터
oedit - 객체 에디터
medit - 몹 (NPC) 에디터
zedit - 구역 에디터
sedit - 상점 에디터
trigedit - 트리거 에디터
```
### 구역 관리
```
zreset - 현재 구역 재설정
saveall - 모든 OLC 변경 저장
show zones - 모든 구역 목록
goto <room#> - 방으로 텔레포트
```
### 관리자 명령어
```
advance <player> <level> - 플레이어 레벨 설정
wizhelp - 불멸자 명령어 목록
shutdown - 서버 종료
copyover - 플레이어 연결 끊기 없이 재부팅
```
## 세계 빌딩
### 첫 번째 구역 생성
1. **구역 생성:**
```
zedit new 100 - 구역 100 생성
zedit 100 name <구역 이름>
zedit 100 top 199 - 방 범위 설정 (100-199)
zedit save
```
2. **방 생성:**
```
redit 100 - 방 100 생성/편집
redit name <방 이름>
redit desc - 방 설명 입력
redit exit north 101 - 북쪽을 방 101로 연결
redit save
```
3. **객체 생성:**
```
oedit 100 - 객체 100 생성/편집
oedit name sword
oedit short a steel sword
oedit long A sharp steel sword lies here.
oedit save
```
4. **몹 (NPC) 생성:**
```
medit 100 - 몹 100 생성/편집
medit name guard
medit short a city guard
medit long A guard watches the area carefully.
medit level 10
medit save
```
5. **모든 것 저장:**
```
saveall - 모든 OLC 작업 저장
```
## 구성
### 주요 구성 파일
- `src/campaign.h` - 핵심 게임 설정
- `src/mud_options.h` - 서버 옵션
- `src/vnums.h` - 가상 번호 할당
- `lib/etc/config` - 런타임 구성
### 포트 변경
시작 명령어 편집:
```bash
./bin/circle -q 5000 -d lib # 포트 5000에서 실행
```
### 데이터베이스 설정 (필수)
MySQL/MariaDB 설정 세부 사항은 [배포 가이드](deployment/DEPLOYMENT_GUIDE.md#database-configuration-required)를 참조하세요.
## 문제 해결
### 일반적인 문제
**연결 불가:**
- 서버 실행 확인: `ps aux | grep circle`
- 방화벽 설정 확인
- `telnet localhost 4000`으로 테스트
**빌드 오류:**
- 모든 종속성 설치 확인
- Makefile 누락 시 `autoreconf -fvi` 실행
- 오류를 위해 `log/syslog` 확인
**파일 누락:**
- 모든 필수 파일 생성을 위해 `./scripts/deploy.sh --auto --init-world` 실행
- 심볼릭 링크 존재 확인: `ls -la world text etc`
